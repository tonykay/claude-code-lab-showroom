CLAUDE.md

CLAUDE.md files enable Claude to load persistent, project-specific instructions automatically at session start.

Key Content Guidelines

Include only info that would cause mistakes if absent:

  • Exact build commands
  • Architectural decisions
  • Coding conventions
  • Environment requirements

Exclude: standard knowledge, obvious reminders, sensitive data, frequently changing info.

File Location Hierarchy

Increasing priority:

  1. System-wide (/etc/claude-code/)
  2. User global (~/.claude/)
  3. Project level (CLAUDE.md or .claude/)
  4. Local private (CLAUDE.local.md, gitignored)
Files are loaded in the following order. Because the model attends more to content that appears later in context, later files have higher effective priority.

Advanced Features

@include Directive

Reference other files with relative, home, or absolute paths.

Frontmatter Rules

Use .claude/rules/ for conditional rule application with glob patterns:

---
paths:
  - "src/api/**"
---

Tools

  • /init generates starter CLAUDE.md
  • /memory opens editor
  • claudeMdExcludes prevents loading from specific directories
