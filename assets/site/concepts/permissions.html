Concepts / Permissions

Permissions

The permission system manages which operations execute automatically vs requiring approval.

Six Permission Modes

ModeBehavior
defaultEvaluates each tool call, prompts for side effects, auto-approves read-only
acceptEditsAuto-approves file modifications, prompts for shell
planAnalysis only, all writes/bash blocked
bypassPermissionsDisables all checks (sandboxed environments only)
dontAskAuto-approving in scripted scenarios
autoExperimental classifier-based mode (feature flag)

Configuration

Set via CLI flag, /permissions command, or settings.json files.

Permission Rules System

Allow/deny lists with tool name, optional pattern matching, and behavior. Evaluated before mode. Rules sourced from 5 locations: user, project, local, session, CLI args.

Bash-Specific Rules

Pattern matching with wildcards; compound commands checked independently; most restrictive result applies.

Safety Recommendations

  • Start with default mode
  • Use plan for unfamiliar codebases
  • Avoid bypassPermissions in interactive contexts
