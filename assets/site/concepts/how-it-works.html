How It Works

Claude Code operates as a terminal-based coding agent that executes a continuous agentic loop.

The Agentic Loop

A six-step cycle:

  1. User Input — Messages entered interactively or via stdin/flags
  2. Context Assembly — System prompt built with git status, date, memory files, and available tools
  3. Model Reasoning — Claude analyzes the task and selects appropriate tools
  4. Permission Evaluation — Tools checked against current permission mode before execution
  5. Tool Execution — Approved tools run and return results to the conversation
  6. Loop Continuation — Model receives results and either calls more tools or finalizes response
The loop runs entirely in your terminal process — with no remote execution.

Context Loading

Two context blocks prepended to every API call:

  • System Context: git info (branch, commits, status) and cache-breaking injections, skipped in remote mode
  • User Context: CLAUDE.md memory files (4-level hierarchy) and current date. Both memoized.

Tool Permissions

Each tool has a checkPermissions method returning one of: allow (runs immediately), ask (prompts user), or deny (rejected with error).

Operating Modes

Interactive REPL with live terminal UI or non-interactive print mode for automation. Sub-agents via Task tool.

Storage and Resumption

JSON transcripts with unique session IDs. Long conversations compacted automatically.
