Memory & Context

Claude Code uses a 4-level memory hierarchy to load instructions from markdown files.

Priority System

Later files have higher precedence:

  1. Managed memory (/etc/claude-code/CLAUDE.md) — System-wide administrator settings
  2. User memory (~/.claude/CLAUDE.md) — Personal global preferences
  3. Project memory (CLAUDE.md, .claude/CLAUDE.md) — Team conventions
  4. Local memory (CLAUDE.local.md) — Private project overrides (git-ignored)
Files closer to the current working directory are loaded later and therefore have higher priority.

File Discovery

Walks from CWD up to filesystem root, collecting at each level. Cached for conversation; use /memory to reload.

@include Directive

Reference other files using @ notation:

@./docs/architecture.md

Supports relative, home, and absolute paths. Circular references are detected; maximum depth is 5.

Granular Rules

Use .claude/rules/*.md with path-scoped frontmatter:

---
paths:
  - "src/api/**"
---

Best Practices

  • Managed: Org policies, security guardrails
  • User: Personal preferences across projects
  • Project: Team conventions (testing, builds, architecture)
  • Local: Personal workflow tweaks

Max recommended: 40,000 characters per file.
