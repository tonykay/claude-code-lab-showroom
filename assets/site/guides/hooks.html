Guides / Hooks

Hooks

Hooks are automated commands that trigger at specific lifecycle events.

Exit Codes

  • 0 = success
  • 2 = block/inject
  • other = error to user

Events

EventDescription
PreToolUseBefore tool execution; exit 2 blocks
PostToolUseAfter successful tools; formatters/linters
StopBefore Claude concludes; exit 2 continues conversation
SessionStartAt session beginning; inject context
UserPromptSubmitOn prompt submission; exit 2 blocks
FileChangedMonitors file patterns

Configuration

Use the /hooks command. Lives in settings files (user, project, local).

Hook Types

  1. Shell command — bash/PowerShell with timeout
  2. HTTP request — POST to endpoints
  3. LLM prompt — Claude for decision-making
  4. Agent hook — Full agent with tool access

Common Uses

  • Auto-formatting
  • Running tests
  • Logging
  • Blocking dangerous commands
  • Injecting environment variables
