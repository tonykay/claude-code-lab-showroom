Guides / Skills

Skills

Skills are reusable markdown-based workflows invoked via slash commands.

Structure

Directory with SKILL.md file containing frontmatter and instructions. Stored in .claude/skills/ directories.

Locations

  • Project: .claude/skills/
  • User: ~/.claude/skills/

Invocation

Use /skill-name with optional arguments ($ARGUMENTS or named vars).

Features

  • Inline shell commands with ! prefix for live project state
  • Conditional activation via paths frontmatter
  • Namespaced: nested dirs create colon-separated commands (/database:migrate)
  • Built-in skills for onboarding, code review, analysis

Frontmatter Options

KeyPurpose
descriptionSkill description
argument-hintUsage hint
allowed-toolsRestrict available tools

modelModel override

contextAdditional context
pathsPath-scoped activation
versionSkill version
hooksLifecycle hooks

Skills vs Hooks

Skills are explicit repeatable workflows; hooks automate side effects.
