Introduction

Claude Code is an AI coding agent that runs in your terminal, reading, editing, and executing code across your entire codebase.

The tool operates as a terminal-based AI agent with direct filesystem, shell, and tool access, enabling users to describe tasks in natural language for end-to-end implementation without manual copying or context switching.

Core Capabilities

  1. File Operations — Reading source files, writing new content, and making targeted edits with diff previews for user control
  2. Shell Execution — Running tests, build scripts, git operations, and shell commands with configurable permission controls
  3. Codebase Search — Finding files via glob patterns and searching content with regular expressions for large codebases
  4. Web Fetching — Pulling documentation, API specs, and web searches without leaving terminal sessions
  5. Multi-Agent Coordination — Spawning sub-agents to handle parallel workstreams on complex problems
  6. MCP Server Integration — Extending capabilities through Model Context Protocol servers for databases and APIs

Permission System

Four permission modes control Claude's autonomy:

  • default — Requires approval for shell commands and edits
  • acceptEdits — Auto-applies file edits; shell commands need approval
  • plan — Produces plans requiring sign-off before action
  • bypassPermissions — Runs all actions without prompts (sandboxed environments only)

Activate via --permission-mode flag or /permissions command.

CLAUDE.md Memory System

Project-specific knowledge files at three levels:

  • Project root (shared, version-controlled)
  • Personal scope (private, gitignored)
  • Subdirectory scope (monorepo-specific)

Run /init to auto-generate optimized memory files.

Authentication

Two methods:

  • OAuth through claude.ai (recommended)
  • ANTHROPIC_API_KEY environment variable (CI/automation)

Cloud provider support available for AWS Bedrock and GCP Vertex AI.
